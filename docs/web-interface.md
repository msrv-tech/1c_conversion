# Веб-интерфейс

Веб-интерфейс это обертка над тем же пайплайном `import -> process -> export`, который доступен через CLI. Он реализован в `web_interface.py` на Flask.

## Что умеет интерфейс

Основные функции:

- показывать список поддерживаемых каталогов из `CONF/catalog_mapping.json`;
- запускать полный обмен по каталогу;
- запускать экспорт по кодам или UUID;
- обновлять версии договоров, если доступен `tools/contract_version_manager.py`;
- показывать логи выполнения;
- отдавать статус фоновых задач;
- показывать содержимое SQLite-баз через встроенный просмотрщик.

## Как запустить

Типовой запуск:

```bash
python web_interface.py
```

После старта Flask поднимает HTML-интерфейс из шаблонов:

- `templates/index.html`
- `templates/db_view.html`

## Откуда интерфейс берет данные

Основные источники:

- `CONF/catalog_mapping.json` для списка каталогов;
- `.env` для списка приемников;
- `BD/` для просмотра сырых и обработанных БД;
- `logs/` для логов фоновых задач.

В коде заранее описан список приемников:

- `default`
- `preprod`
- `matveev`
- `prod`

Для каждого проверяется наличие соответствующей переменной окружения.

## Как запускаются операции

Веб не выполняет бизнес-логику внутри Flask-роутов напрямую. Вместо этого:

- собирается команда запуска Python-скрипта;
- операция уходит в отдельный поток;
- stdout и stderr пишутся в лог;
- фронт читает статус и лог через polling API.

Такой подход упрощает сопровождение, потому что веб использует тот же CLI-контур, что и терминальные запуски.

## Основные маршруты

Ключевые endpoints в `web_interface.py`:

- `/` - главная страница;
- `/api/catalogs` - список каталогов;
- `/api/run/<catalog_name>` - запуск обмена по каталогу;
- `/api/logs/<catalog_name>` - чтение лога;
- `/api/status/<catalog_name>` - статус выполнения;
- `/api/export-by-codes/<catalog_name>` - точечный экспорт по кодам или UUID;
- `/api/update-contract-versions` - обновление версий договоров;
- `/api/connections` и `/api/targets` - информация о подключениях;
- `/db/view/<catalog_name>` - просмотрщик БД;
- `/api/db/tables/<catalog_name>` - список таблиц в БД;
- `/api/db/table/<catalog_name>/<table_name>` - содержимое таблицы.

## Просмотр БД

Интерфейс умеет открывать как сырую, так и обработанную БД каталога.

Это удобно для:

- сверки результатов импорта;
- проверки обработки перед экспортом;
- поиска аномалий без отдельного SQLite-клиента;
- поддержки аналитиков и разработчиков без прямой работы в консоли.

## Ограничения и практические замечания

- веб зависит от тех же прав доступа, что и CLI;
- COM-операции все равно выполняются на той машине, где запущен Python;
- для стабильной работы важны корректные строки подключения в `.env`;
- экспорт по кодам работает только из обработанной БД;
- `prod` в интерфейсе должен использоваться отдельно и осознанно, так как он переключает экспорт на продовый приемник.

## Когда использовать веб, а когда CLI

Веб подходит для:

- повторяющихся операционных запусков;
- просмотра БД;
- адресного экспорта по кодам или UUID;
- удаленного мониторинга логов внутри команды.

CLI удобнее для:

- разработки новых loader/processor/writer;
- детальной отладки и профилирования;
- массовых scripted-запусков;
- обновления метаданных и сервисных операций.
