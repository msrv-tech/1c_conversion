# Импорт

Импорт в проекте означает чтение данных из 1С источника и сохранение их в SQLite. За эту стадию отвечает `main.py --import` и загрузчики из папки `IN/`.

## Как это устроено

Поток выполнения:

1. `main.py` определяет каталог и путь к сырой БД.
2. `load_stage.py` выбирает соответствующий модуль `IN/<catalog>_loader.py`.
3. Загрузчик подключается к 1С через общий COM-слой из `tools/onec_connector.py`.
4. Полученные строки сохраняются в SQLite.
5. Если загрузчик объявляет `TABLE_NAME`, после импорта может выполняться автоматическое обновление типов перечислений через `tools/update_reference_types.py`.

По умолчанию для каждого каталога создается отдельная база:

- `BD/<catalog>.db` для сырого импорта;
- исключение: метаданные `configuration_structure` обычно пишутся в `CONF/*.db`.

## Основные флаги CLI

- `--import` включает стадию импорта.
- `--catalog` задает каталог, например `contractors`.
- `--source-1c` указывает источник 1С.
- `--sqlite-db` задает корень каталога БД или путь к файлу для специальных сценариев.
- `--mode test|full|incremental` управляет объемом загрузки.
- `--filters-db` подключает SQLite с пользовательскими фильтрами UUID.
- `--json-output` используется только там, где загрузчик умеет писать JSON, в первую очередь для метаданных.
- `--metadata-db` задает базу метаданных для последующего обновления типов.
- `--skip-type-update` отключает постобработку типов перечислений.
- `--verbose` включает подробный лог.

## Режимы загрузки

- `test`: первые 50 строк, удобно для проверки загрузчика и схемы.
- `full`: полная выгрузка.
- `incremental`: пропускает каталог, если соответствующая SQLite-база уже существует.

Важно: в `main.py` режим `incremental` не догружает частично отсутствующие записи внутри файла, а именно скипает уже существующую БД каталога.

## Типовой запуск

Импорт одного справочника:

```bash
python main.py --import --catalog contractors --source-1c source --sqlite-db BD --mode test
```

Импорт с фильтрами:

```bash
python main.py --import --catalog nomenclature --source-1c source --sqlite-db BD --filters-db BD/filters.db --mode full
```

Импорт всех доступных каталогов:

```bash
python main.py --import --catalog all --source-1c source --sqlite-db BD --mode full
```

## Что лежит внутри сырой БД

В каждой БД обычно есть одна основная таблица с именем каталога, например:

- `contractors` в `BD/contractors.db`
- `nomenclature` в `BD/nomenclature.db`

При импорте поля ссылочного типа и перечисления обрабатываются так:

- ссылки сохраняются как JSON со служебными полями `presentation`, `uuid`, `type`;
- перечисления стараются нормализоваться в строку вида `Перечисление.<Имя>.<Значение>`.

Эта логика вынесена в `tools/db_manager.py` и `tools/onec_connector.py`.

## Связь с catalog mapping

`CONF/catalog_mapping.json` не управляет самим вызовом `main.py --import`, но фиксирует рабочую маршрутизацию проекта:

- какое русское имя объекта 1С соответствует `catalog_name`;
- какой loader/process/writer используется;
- какой тип загрузки ожидается;
- какие табличные части участвуют.

На практике это главный реестр поддерживаемых объектов для команды и для веб-интерфейса.

## На что смотреть при отладке импорта

- существует ли модуль `IN/<catalog>_loader.py`;
- корректна ли строка подключения к источнику;
- создалась ли SQLite-база в `BD/`;
- появилась ли основная таблица каталога;
- обновились ли значения типов перечислений после импорта.

Если импорт завершился, а данных нет, сначала проверяют:

- SQL-запрос загрузчика;
- ограничения фильтров;
- режим `test/full`;
- содержимое `filters.db`, если он передан.

## Связанные файлы

- `main.py`
- `load_stage.py`
- `IN/`
- `tools/db_manager.py`
- `tools/onec_connector.py`
