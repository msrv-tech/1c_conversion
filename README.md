# 1C SQLite Bridge

Репозиторий для миграции данных между базами 1С через промежуточные SQLite-базы и Python.

Основной и наиболее проработанный сценарий этого проекта: миграция из `УПП` в `УХ`. Именно под этот маршрут в репозитории уже лежат:

- метаданные источника `УПП`
- метаданные приемника `УХ`
- готовые JSON-маппинги
- `type_mapping.db`
- наборы `IN / PROCESS / OUT`, развивавшиеся под этот кейс

При этом архитектура не завязана только на `УПП -> УХ`. Репозиторий можно использовать и для других миграций между конфигурациями 1С, если подготовить собственные:

- файлы загрузки в `IN`
- файлы обработки в `PROCESS`
- файлы выгрузки в `OUT`
- метаданные и маппинги в `CONF`

## Как устроен процесс

Общий конвейер такой:

1. выгрузка структуры и данных из базы-источника;
2. сохранение данных в SQLite;
3. преобразование данных и ссылок;
4. загрузка результата в базу-приемник через COM.

Это делает репозиторий удобным для:

- контролируемой миграции крупных справочников и документов
- анализа и отладки данных на промежуточном SQLite-слое
- поэтапного построения маппингов между конфигурациями 1С

## Что уже есть в репозитории

- `CONF/upp_metadata.json`: метаданные источника `УПП`
- `CONF/uh_metadata.json`: метаданные приемника `УХ`
- `CONF/type_mapping.db`: основная база маппинга
- `CONF/type_mapping.json`: выгрузка маппинга в JSON
- `CONF/catalog_mapping.json`: маршрутизация каталогов между стадиями
- `CONF/chart_of_accounts_mapping.json`: маппинг плана счетов

То есть проект уже публично позиционируется не как абстрактный пустой framework, а как рабочая база под `УПП -> УХ`, которую можно переиспользовать и адаптировать.

## Требования

- Python 3.10+
- Windows
- установленная платформа 1С
- доступ к COM (`pywin32`)

Установка:

```bash
pip install -r requirements.txt
```

## Быстрый старт для УПП -> УХ

1. Скопируйте `.env.example` в `.env`.
2. Заполните строки подключения к источнику и приемнику.
3. Используйте уже включенные файлы из `CONF`.
4. Выполняйте импорт, обработку и экспорт через `main.py`.

Примеры:

```bash
python main.py --import --catalog contractors --source-1c source --sqlite-db BD
python main.py --process --catalog contractors --sqlite-db BD
python main.py --export --catalog contractors --target-1c target --sqlite-db BD
```

## Адаптация под другие конфигурации

Если вы хотите использовать этот репозиторий не для `УПП -> УХ`, а для другой пары конфигураций, вам потребуется:

1. выгрузить свои метаданные источника и приемника;
2. собрать свои `CONF/*.json` и `CONF/*.db`;
3. реализовать или адаптировать загрузчики в `IN`;
4. реализовать или адаптировать процессоры в `PROCESS`;
5. реализовать или адаптировать выгрузчики в `OUT`.

Иными словами, сейчас репозиторий универсален по архитектуре, но предметно оптимизирован под сценарий `УПП -> УХ`.

## Структура репозитория

- `IN/`: чтение данных из 1С
- `PROCESS/`: преобразование данных между моделями источника и приемника
- `OUT/`: запись данных в 1С
- `tools/`: общие утилиты для COM, SQLite, маппинга и логирования
- `CONF/`: метаданные и маппинги
- `BD/`: рабочие SQLite-базы
